Communicon

WEB APPLICATION

기획 및 스토리

웹 공부를 시작하였을 때, 자바스크립트를 공부하면서 프레임워크에 대해서 알게 되었습니다. 그중에 풀스택 프레임워크인 MeteorJS라는 것이 눈에 띄었고, 튜토리얼을 따라 하면서 어떻게 개발할 것인지 익혀갔습니다. 당시 Meteor를 작성하는 프론트엔드 프레임워크로는 AngularJs(angular 1)과 reactJS 그리고 Blaze가 있었는데 초심자는 Blaze가 쉽다는 말에 Blaze로 시작을 하게 되었습니다. 그리고 튜토리얼을 5번 정도 반복한 뒤 익숙해졌을 무렵 교내 아이디어 경진대회에서 재밌어 보이는 아이디어를 발견하였는데 그것이 바로 Communicon이라는 아이디어였습니다.
말이 통하지 않는 지역에서 아이콘을 짚어서 대답을 얻는 티셔츠에서 영감을 받은 아이디어로 아이콘을 선택해서 언어가 다른 상대방에게 간단한 질문을 할 수 있는 애플리케이션을 개발하기 시작하였습니다.

Programing with : MeteorJs(Blaze) / Bootstrap

Period : 2016(3 days)

Code : github

내용 소개

아이콘으로 대화를 해보자는 아이디어를 구체화한 웹 애플리케이션입니다. 아이콘을 선택하여 질문 아이콘을 만들고 답변을 얻어내는 애플리케이션입니다. MeteorJS 프레임워크를 Blaze로 개발하고, CSS프레임워크인 BootStrap을 사용하여 화면을 꾸몄습니다.

Communicon의 메인 화면. 질문할 아이콘을 선택할 수 있다.

Communicon의 질문 화면. 선택된 아이콘으로 상대방에게 질문하는 화면이다.

Communicon의 답변 화면. 선택된 메인아이콘에 따라 답변 화면이 다르게 적용된다. 위치에 대한 답변화면.

Communicon의 언어 선택 화면. 일본어 이탈리아어 영어가 지원된다. 아이콘에 대한 답변요청과 감사인사의 언어가 바뀐다.

Communicon의 일본어를 적용한 질문 화면.

Communicon의 일본어를 적용한 답변화면. 가격에 대한 답변화면.

배운 점 & 부족했던 점

처음으로 개발해본 웹 애플리케이션이었습니다. 처음으로 BootStrap을 써보고 신세계를 느끼기도 하였고, 튜토리얼에서 배운 모든 것을 써볼 수 있어서 아주 좋은 경험이었다고 생각합니다. 개발이 너무 재밌어서 며칠 밤을 새우며 완성하였습니다.
사실 아이디어의 한계가 많았습니다. 같은 상황에서 아이콘보다 효율적인 방법이 많이 있었기 때문입니다. 그렇기 때문에 아쉽게도 해커톤 경진대회에서는 본선진출까지만 할 수 있었습니다. 하지만 아이디어의 한계를 극복할 수 있는 UX를 적용한다면 실사용이 가능할 것으로 생각합니다.
